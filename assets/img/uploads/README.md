# Папка для загруженных фото

Эта папка предназначена для хранения фотографий, загруженных через админ-панель:
- Логотип компании
- Фото главного экрана (hero)
- Фото карточек товаров
- Обложки статей блога и фото внутри статей
- Фото галереи
- Фото блока «О нас»
- Фото блока «О товаре»
- Фото на фон блоков

## Как это работает

**Сейчас (без сервера):** все загруженные через админку файлы сохраняются в браузере в `localStorage` в формате `data:image/...;base64,...`. Это работает только в рамках одного браузера и одного устройства.

**Для продакшена (с сервером):** нужно подключить backend (PHP/Node.js), который:
1. Принимает файл через `<input type="file">` или FormData
2. Сохраняет в папку `/assets/img/uploads/`
3. Возвращает URL вида `assets/img/uploads/photo-123.jpg`
4. Админка сохраняет URL в `localStorage`, а не base64

## Именование файлов

При ручной загрузке фото используйте понятные имена:
- `logo.png` — логотип
- `hero-main.jpg` — главное фото
- `product-gazoblok-300.jpg` — карточка товара
- `blog-article-title.jpg` — обложка статьи
- `gallery-01.jpg`, `gallery-02.jpg` — галерея
- `about-factory.jpg` — блок «О нас»

## Поддерживаемые форматы

- **JPG/JPEG** — фото (оптимально для большинства случаев)
- **PNG** — логотипы, прозрачные изображения
- **WebP** — современный формат, меньший размер
- **SVG** — векторные иконки, логотипы
- **MP4** — видео на фон (не храните в этой папке, используйте отдельную `/assets/video/`)

## Рекомендуемые размеры

- Логотип: 300×100 px (3:1)
- Hero фото: 1280×1280 px (1:1)
- Карточка товара: 800×600 px (4:3)
- Обложка блога: 1200×675 px (16:9)
- Галерея: 1200×800 px (3:2)
- Фон блока: 1920×1080 px (16:9)

# Руководство по админке EcoStroyDom

Админка — это простая веб-страница, которая редактирует любой текст, цену, фото и карточку сайта напрямую в репозитории GitHub. Никакого бэкенда, CMS или базы данных — всё хранится в JSON-файлах, HTML-страница патчится через GitHub API.

---

## Вход

1. Откройте **https://ecostroydom.kz/admin.html**
2. Вставьте GitHub Personal Access Token (PAT) в поле и нажмите «Войти».
3. Токен сохраняется в браузере локально (`localStorage`) — повторный вход не нужен.

### Как получить токен

- Перейдите на https://github.com/settings/personal-access-tokens/new
- Тип: **Fine-grained personal access token**
- Repository access: **Only select repositories** → `779722-hub/ecostroydom`
- Permissions → Repository → **Contents: Read and write**
- Сгенерировать → скопировать (токен показан один раз!)

**Важно:** никогда не храните токен в файлах внутри репозитория и не показывайте его никому. Если утёк — немедленно Revoke в настройках GitHub и сгенерируйте новый.

## Вкладки админки

### 🔥 Акция
Редактирование плашки «Предзаказ» на главной:
- Заголовок, подзаголовок, описание, бейдж-ценник
- Дата и время окончания (для таймера обратного отсчёта)

Сохраняется в `assets/js/config.js`.

### 🎨 Темы
5 параллельных цветовых схем (слотов). Каждый слот содержит палитры для тёмной и светлой темы:
- **accent** — главный акцентный цвет (заголовки, кнопки)
- **bg / bg2 / surface / surface2** — фоны
- **text / textDim** — цвета текста
- **line / mintGlow** — второстепенные

Активный слот применяется ко всему сайту через CSS-переменные.

### 📝 Тексты
Все уникальные тексты сайта разбиты по группам:
- Hero (верхняя секция): заголовки, подзаголовки, цена, бейджи
- Stats (метрики)
- Sections (заголовки секций)
- About (о компании)
- Footer (подвал, адрес, примечание)
- SEO-блок

Редактирование текста обновляет и JSON, и HTML в одном коммите — поисковики видят актуальный контент сразу.

### 📦 Карточки
Подтабы для всех карточных блоков сайта:
- **Каталог** — 6 товаров газоблока
- **Сопутствующие** — кирпич, пенобетон, бетон, термопанели
- **Доставка** — тарифы манипулятора / самосвала
- **Преимущества** — 6 блоков с иконками
- **Шаги** — процесс заказа (6 шагов)
- **FAQ** — вопросы и ответы
- **Галерея** — фото работ
- **Отзывы** — по 3 колонкам со скроллом
- **Блог** — карточки статей

Для каждой карточки: редактирование, добавление, удаление, перестановка ↑↓.

### 💰 Цены
Единая таблица всех цен с сайта: каталог + сопутствующие + доставка. Можно массово поднимать/опускать или править по одной.

### 📸 Фото
Загрузка новых фото (автоматическая компрессия до ~500 КБ, JPEG 85%). Список всех изображений, замена по клику.

## Редактирование статей блога

### Редактировать существующую

1. Карточки → подтаб **Блог** → «Редакт.» у нужной статьи
2. Меняйте мета-поля (заголовок, категория, дата, фото, excerpt) — они обновят карточку на главной и в `/blog/`
3. Нажмите **📥 Загрузить текущее содержимое** — загрузится HTML тела статьи
4. Отредактируйте HTML в большом поле (поддерживаются `<h2>`, `<p>`, `<ul>`, `<table class="article__table">`, `<img class="inline">`, `<b>` и любые другие теги)
5. **👁 Предпросмотр** для визуальной проверки
6. **Применить** → **Сохранить и опубликовать**

### Создать новую статью

1. Карточки → Блог → **+ Новая статья**
2. Придумайте заголовок — slug (имя файла) сгенерируется автоматически
3. Заполните дату, категорию, фото обложки, excerpt
4. В поле «Содержимое» замените стартовый HTML на текст статьи
5. **Применить** → **Сохранить и опубликовать**
6. Через ~1 минуту статья будет доступна по адресу `ecostroydom.kz/blog/<slug>.html`

### Удалить статью

Удаление из списка в админке убирает карточку с главной и со страницы `/blog/`, но **не удаляет сам HTML-файл** статьи. Если нужно физическое удаление файла — запросите у разработчика.

## Как работает сохранение

При нажатии «Сохранить и опубликовать»:
1. Админка собирает все изменённые файлы в один атомарный коммит через GitHub Git API
2. Коммит пушится в ветку `main` репозитория `779722-hub/ecostroydom`
3. GitHub Pages автоматически пересобирает сайт (~1 минута)
4. Обновите страницу сайта через Ctrl+F5

**В одном коммите** могут быть: JSON-файлы данных, `index.html` (пропатчена между маркерами), `blog/index.html`, конкретные HTML-статьи — всё в одной транзакции.

## Если что-то сломалось

### Ошибка «Маркеры не найдены»
Кто-то руками удалил HTML-комментарии вида `<!-- CARDS:catalog START -->` в `index.html`. Восстановите из истории Git или обратитесь к разработчику.

### Конфликт при сохранении
Если параллельно кто-то пушил в репозиторий (или вы сами через deploy.bat), админка может показать ошибку SHA. Решение: перезагрузите `/admin.html` (F5), ваши несохранённые изменения пропадут — сделайте их заново.

### Сайт не обновился через минуту
Проверьте билд на https://github.com/779722-hub/ecostroydom/actions — красный крестик значит, что сборка упала. Обычно из-за синтаксической ошибки в только что отредактированном HTML (незакрытый тег).

### Откат изменений
Открыть https://github.com/779722-hub/ecostroydom/commits/main, найти плохой коммит, нажать «Revert». GitHub создаст обратный коммит автоматически.

## Техническая архитектура (кратко)

- **Данные:** JSON-файлы в `data/` (content, catalog, upsell, delivery, features, steps, faq, gallery, testimonials, blog-posts, themes)
- **HTML:** `index.html`, `blog/*.html` — патчатся между маркерами `<!-- CARDS:X START/END -->` и `<!-- CONTENT:key START/END -->`
- **Рендеры:** `assets/js/content-renderer.js` + `theme-renderer.js` применяют JSON к DOM на клиенте (для быстрого обновления после смены темы, не критично для SEO)
- **Stack:** pure HTML/CSS/vanilla JS, GitHub Pages, без фреймворков и билд-систем
